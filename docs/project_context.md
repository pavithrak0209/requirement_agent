## Conventions for Naming Tasks

### Task ID Format
Tasks are auto-generated with the prefix `SRC-` followed by a zero-padded 3-digit number (e.g., `SRC-001`, `SRC-002`).

### Task Types
- **bug**: A defect, regression, or unexpected behaviour found in the system.
- **story**: A user-facing feature or capability described from the user's perspective.
- **task**: A technical or operational work item that is not directly user-facing.
- **subtask**: A smaller unit of work that is part of a larger story or task.

### Task Heading Guidelines
- Keep headings short and action-oriented (e.g., "Fix null pointer on login", "Add CSV export to reports").
- Start with an imperative verb: Fix, Add, Update, Remove, Migrate, Investigate, etc.

### Description Guidelines
- Provide full context: what is expected, what is observed (for bugs), or what needs to be built.
- Include acceptance criteria where possible.

### Location Field
- For document sources: include page number or section heading (e.g., "Page 4 — Section 3.2").
- For video/audio transcripts: include timestamp (e.g., "00:14:32").
- If no location is determinable, leave as `null`.
